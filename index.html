git 简单使用指南

一、安装 git

对于 windows 用户，到官网链接：https://gitforwindows.org/ 下载 ，使用” next “安装即可。

二、git 主要功能：本地仓库与远程仓库的交互

本地仓库和远端仓库的文件上传和下载，重复的、未更新的文件不会被上传。

本地仓库上传到远程仓库

  • 创建一个文件夹，作为我们的workspace，即初始化我们的本地仓库

创建同目录下的.git 文件，有了这个文件才有后续的操作

  • 将自己需要上传的文件，先放置缓存区staging area
    缓存具体文件

缓存目录下所有文件

查看文件状态

  • 将缓存文件提交到本地仓库local repository，才能推到远端仓库remote repository

    message 是通过 -m 参数添加的提交信息，

  • 推到远程仓库的前一步，需要添加远程仓库地址

    把 origin 关联远程仓库地址，origin 是可以换成你喜欢的名字

查看是否关联

  • 上传远程仓库 git 所关联的远程仓库地址的其他分支都是 ok 的。

远程仓库下载到本地仓库

  • 克隆远程仓库的

  • 如果要上传本地仓库中的修改后的文件，或者新添加的文件，与 上述本地仓库上传到远程仓库步骤基本一致，去除第一步和第四步即可

  • 由于 git clone 命令会产生一个 .git 文件，并且 git remote -v 查看已经关联本地和远程了。

分支操作

默认分支是master分支上，如果你的远程仓库具有多个分支

  • 将本地仓库中的文件上传到远程已有的 otherbranch分支

  • 执行git push -u origin otherbranch就会向你远程仓库的otherbranch分支上传你的文件，而且只能在远程otherbranch分支上看到你刚才上传的文件。

  • 如果远程仓库没有此分支，在你git push后，会自动在远程仓库中添加一个此分支的同名分支。

基于 Mac，使用 hexo-cli 创建自己的第一个博客

一、安装及运行步骤

去 node 官网 ：https://nodejs.org/ 下载（最新的就好），使用“next”安装

在 mac 终端查看

npm 速度较慢，使用国内的淘宝镜像

下载 hexo 脚手架，如果用过 vue 的同学肯定知道 vue 脚手架，同理下载 hexo 脚手架

成功后，体验 hexo 博客

markdown 语法知识，可参见菜鸟教程-markdown

非本地访问，需要借助 github 仓库

  • 先去 GitHub 上创建一个仓库，这个很简单，只不过仓库名需要以下面这种方式命名
  • 修改配置文件_config.yml，git 方式（type）部署（deploy）到仓库（地址为 repo）的分支（branch）master 中

正式访问

使用https://YouGithubName.github.io访问你的博客网站

二、安装遇到的错误

  1. hexo d 出错，再次访问 github，提示我出发滥用机制，禁止访问，几分钟甚至几个小时后再来访问

这个问题我百度了一圈也没找到有效的解决方法，最后硬核破解，直接把工程目录删除！！！重建一个目录，安装 hexo-cli 脚手架。

  1. hexo init 出错

这个问题，我在换了个网的时候成功 init，此时 github 也能正常访问

  1. 无法访问 YourGithubName.github.io

这个问题，可以稍微等一段时间，需要等待程序反应时间，这个时间有一点长

三、总结

第一个博客，就这样踉踉跄跄的搭建成功了，这也是我在这个博客系统上写的第一篇博客，不断产出，不断进步。
